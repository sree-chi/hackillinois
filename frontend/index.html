Policy Engine

Never let an AI agent drain your bank account. Define strict JSON policies that enforce maximum spend limits, allowed HTTP methods, and rate limits. If an agent hallucinates or goes rogue, Sentinel blocks the request before it reaches your real APIs.

Web3 Proofs

Solana-Backed Audits

Trust but verify. Every decision Sentinel makes—whether an action is allowed or blocked—is cryptographically hashed and anchored to the Solana devnet. You get a permanent, immutable Web3 receipt proving exactly what the AI intended to do.

High-Risk Intercept

The 402 Payment Required Flow

Some actions are too expensive to trust to AI alone. If an agent requests an action that exceeds your high-risk threshold (e.g., spending over $1,000), Sentinel automatically intercepts it with an HTTP 402 Payment Required challenge.

The agent is temporarily blocked and must provide a valid cryptographic signature (an x402 token) from an authorized developer's Solana wallet before the action is unlocked and executed.

Developer Experience

Built for Agentic Workflows

Easily integrate Sentinel into your OpenAI, Anthropic, or Gemini tool-calling loops. Ask for permission first, execute second.

agent-execution-loop.js 
// 1. Agent intercepts intent and asks Sentinel for permission
const auth = await fetch('https://api.sentinel-auth.com/v1/authorize', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${API_KEY}` },
  body: JSON.stringify({
    policy_id: "pol_strict_financial",
    action: { type: "wire_transfer", amount_usd: 50.0 },
    reasoning_trace: "Transferring funds as requested by user."
  })
});

// 2. Sentinel validates the policy and anchors to Solana
const { allowed, receipt_signature } = await auth.json();

if (allowed) {
  // 3. Action is safe. Execute the real financial API!
  executeWireTransfer(receipt_signature);
}
