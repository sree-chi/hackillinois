Overview

What is Sentinel Auth?

Authorize · Audit · Secure · Anchor

Sentinel Auth is a REST API and policy engine that sits between your autonomous AI agents and real-world APIs. You define strict security policies (e.g., maximum spend limits, allowed HTTP methods, and rate limits), and your AI agents must request permission via Sentinel before acting.

Sentinel detects policy violations, strictly limits financial exposure, and provides a Solana-backed Web3 audit trail by anchoring every allowed/blocked action to the blockchain. High-risk actions automatically trigger a 402 Payment Required challenge, requiring a cryptographic signature (x402 token) from an authorized wallet before proceeding.

Core API

Authorization Engine

The core engine. Submits an agent's intended action for evaluation against a specific policy.

POST /v1/authorize

Request Body

Field Type Req Description
policy_id string yes The ID of the policy to evaluate against.
requester string yes Identity of the agent (e.g., agent://trading_bot).
action.type string yes Short action category (e.g., wire_transfer).
action.http_method string yes HTTP method (e.g., POST, GET).
action.resource string yes Target resource path/URI.
action.amount_usd number no Estimated dollar cost of the action.
reasoning_trace string yes AI's chain-of-thought justifying the action.
Response: 200 OK (Approved)
{
  "request_id": "req_12345...",
  "policy_id": "pol_a1b2c3d4...",
  "action_hash": "e4d90f...",
  "allowed": true,
  "decision": "allow",
  "receipt_status": "anchored",
  "receipt_signature": "5kG9X...solana_tx...",
  "created_at": "2025-01-15T12:05:00Z"
}
Response: 403 Forbidden (Blocked)
{
  "error": {
    "type": "policy_error",
    "code": "POLICY_LIMIT_EXCEEDED",
    "message": "Action exceeds budget.",
    "status": 403,
    "details": {
      "safety_violation": {
        "category": "spend_limit_exceeded"
      }
    }
  }
}
Response: 402 Payment Required (High Risk)
{
  "detail": "High-risk actions require a verified payment token via x-solana-tx-signature."
}

Triggered automatically if amount_usd is > $1,000. Agent must resubmit the exact same payload but include a signed micro-transaction in the x-solana-tx-signature header.

Reference

Request Format

Every agent request must explicitly state what it intends to do before it does it.

Action Schema

Submitted within the body of a POST request to /v1/authorize.

policy_id string ID of the ruleset to evaluate against
requester string A string identifying the agent
action.type string Short action categorization (e.g. api_call)
action.http_method string GET, POST, PUT, DELETE
action.resource string URI or path the agent intends to interact with
action.amount_usd number Dollar amount this action will cost/move
reasoning_trace string LLM chain-of-thought justifying the action

Reference

Limits & Errors

Per-Policy Budgets: Enforced by comparing the sum of all previously approved amount_usd for a policy against max_spend_usd. Returns 403 Forbidden if exceeded.

Per-Policy Rate Limits: Controlled by max_requests_per_minute. Returns 429 Too Many Requests if exceeded.

High-Risk Auth (x402): Actions requesting amount_usd >= $1,000 automatically fail with 402 Payment Required unless a valid Solana signature is attached.

Standard Error Envelope
{
  "detail": {
    "error": {
      "type": "policy_error",
      "code": "RATE_LIMIT_EXCEEDED",
      "message": "The maximum number of requests per minute for this policy has been exceeded.",
      "policy_id": "pol_a1b2c3d4...",
      "status": 429,
      "details": { "limit": 60 }
    }
  }
}

Example Workflow

Create a Key → Create a Policy → Request Authorization → Execute

End-to-End Terminal Commands
# 1. Get an API key using your Dashboard Session Token
curl -s -X POST http://localhost:8000/v1/developer/keys \
  -H "X-Session-Token: sess_tok_abc..." \
  -d '{"app_name":"Demo Agent"}'

# 2. Create a policy with a $50 spend limit
curl -s -X POST http://localhost:8000/v1/policies \
  -H "Authorization: Bearer ska_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Demo Policy",
    "rules": {"max_spend_usd": 50.0}
  }'

# 3. Agent requests permission to spend $10 on an API call
curl -s -X POST http://localhost:8000/v1/authorize \
  -H "Authorization: Bearer ska_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "policy_id": "pol_your_policy_id",
    "requester": "agent://demo",
    "action": {
      "type": "llm_query",
      "http_method": "POST",
      "resource": "gemini-2.5-flash",
      "amount_usd": 10.0
    },
    "reasoning_trace": "Need to generate text for user."
  }'

# 4. Agent receives 200 OK + Solana receipt signature.
# Agent can now safely make the actual HTTP call to the requested API!
